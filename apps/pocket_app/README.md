# pocket_app — Flutter app (iOS · Android · Web)

This is the new Pocket Budget client. It already builds a runnable dashboard
that derives balances + net worth from the shared, pure `pocket_domain`
package — proving the architecture (presentation → Riverpod → domain) before
the data layer lands.

## First-time setup (run on your machine — needs the Flutter SDK)

The `lib/`, `pubspec.yaml`, and config are committed here, but the native
platform folders (`android/`, `ios/`, `web/`) are generated by Flutter. From
this directory:

```bash
# 1. generate the native shells (won't overwrite the existing lib/ or pubspec)
flutter create --platforms=android,ios,web --org com.mshshakir.pocket --project-name pocket_app .

# 2. resolve dependencies (pulls in the path-linked pocket_domain package)
flutter pub get

# 3. run it
flutter run -d chrome        # web
# or: flutter run             # a connected device / emulator
```

You should see a "Net worth" card and two accounts (a USD and a KES account)
with balances computed by `LedgerMath`, including the KES→USD conversion in the
net-worth total.

## Layout (clean / feature-first architecture)

```
lib/
  main.dart                      ProviderScope + runApp
  app.dart                       MaterialApp.router (theme + routing)
  core/                          theme, router (cross-cutting)
  features/
    dashboard/
      application/               Riverpod providers (use cases / state)
      presentation/             screens + widgets
      domain/   data/           (added per feature as needed)
```

Business logic is **not** here — it lives in `packages/domain` (pure Dart,
shared by every platform). The temporary in-memory providers in
`features/dashboard/application/` will be replaced by repository-backed ones
(`packages/data`, Drift + Supabase + PowerSync) with no change to the widgets.
